
# ПП.02 — Производственная практика (Системное администрирование)

## Описание

Данный репозиторий содержит материалы производственной практики ПП.02 по направлению **«Системное администрирование»**. В ходе практики выполнялись задачи по администрированию информационной системы транспортной компании **ООО «АвтоЛогистик»**, автоматизации работы системного администратора и обеспечению стабильной и безопасной работы компьютерной инфраструктуры предприятия.

Практика выполнялась в среде Windows с использованием PowerShell и средств администрирования операционной системы.

---

## Содержание репозитория

### `docs/`
Документация по практике:
- Отчёт по производственной практике
- Описание предметной области «Транспортная компания»
- Техническое задание на разработку элементов ИС
- UML-диаграммы (Use Case, Class, Sequence, Activity, State, Deployment)

### `scripts/`
Скрипты администрирования PowerShell для автоматизации работы транспортной компании:

| Скрипт | Назначение |
|--------|------------|
| `backup_tms_data.ps1` | Резервное копирование базы данных TMS (Transport Management System) |
| `restore_tms_data.ps1` | Восстановление данных TMS из резервной копии |
| `create_driver_account.ps1` | Создание учётной записи для нового водителя в домене |
| `disable_employee_account.ps1` | Блокировка учётной записи уволенного сотрудника |
| `server_monitoring.ps1` | Мониторинг состояния серверов и служб TMS |
| `disk_space_check.ps1` | Проверка свободного места на серверах и рабочих станциях |
| `network_connectivity_check.ps1` | Проверка сетевой доступности транспортных терминалов и складов |
| `update_terminal_clients.ps1` | Централизованное обновление ПО на терминалах диспетчеров |
| `cleanup_logs.ps1` | Очистка устаревших лог-файлов системы |
| `check_antivirus_status.ps1` | Проверка статуса антивирусной защиты на всех ПК |
| `parse_event_logs.ps1` | Парсинг системных журналов для выявления ошибок |

---

## Цель работы

Целью производственной практики являлось получение практических навыков системного администрирования в условиях реально действующей транспортной компании, автоматизация задач обслуживания ИТ-инфраструктуры, обеспечение безопасности данных о перевозках и повышение надёжности работы информационной системы управления транспортом.

---

## Используемые технологии

- **Windows Server / Windows 10/11**
- **PowerShell** (скрипты автоматизации)
- **UML** (моделирование в PlantUML / Draw.io)
- **Git / GitHub** (управление версиями)
- **TMS (Transport Management System)** — администрируемая система

## Результаты

В ходе практики:

- изучены задачи системного администратора в логистической компании;
- разработаны скрипты автоматизации для типовых задач (бэкап TMS, управление пользователями, мониторинг);
- реализована система резервного копирования критичных данных о рейсах и заказах;
- настроен мониторинг серверов и сетевого оборудования;
- выполнено моделирование информационной системы с помощью UML-диаграмм;
- оформлен отчёт по практике с описанием инфраструктуры и выполненных работ;
- создан публичный репозиторий с результатами работы.
